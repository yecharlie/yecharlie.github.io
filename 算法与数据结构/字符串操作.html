字符串操作

1. 基本方法

1.1 Robin-Karp 子串匹配算法

核心思想是通过对子串快速编码，从而利用编码替代对应子串进行匹配。Robin-Karp算法可以在\(O(N)\)时间内完成子串编码，N为原串长度。设编码基数\(b\)，子串长度为\(n\)，第一个子串处在0位置，根据阅读习惯，子串头部为高位，尾部为低位，于是可以计算子串\(S_0\)的编码\(h_0\)为： \[ h_0=c_nb^{n-1}+c_{n-1}b^{n-2} + \cdots +c_1 \] 这里把\(S_0\)从高位到低位的字符依次命名为\(c_n,\cdots,c_1\)。根据\(S_1\)和\(S_0\)的位置关系，列出\(h_1\)的计算式： \[ h_1=c_{n-1}b^{n-1} + \cdots +c_1b+c_0 \] 其中\(c_0\)是靠近\(c_1\)右侧的字符。可以根据\(h_0\)计算\(h_1\)： \[ \begin{equation} h_1 = (h_0 - c_nb^{n-1})b+c_0 \tag{1} \label{eq:hash} \end{equation} \]

注意到公式\(\ref{eq:hash}\)中的\(h_i\)也适用于\(i > 1\)的情况，此时\(c_n\)表示子串高进位的编码，\(c_0\)则为子串低位编码，\(h_0\)为按顺位前一子串的哈希值。

接下来考虑一些细节问题，使用上述算式得到哈希值往往会很大，所以需要对运算取模，由于子串数目不多于N，为了尽量减少哈希碰撞，根据生日悖论，模数在量级上不应该小于\(\sqrt{N}\)。一般选取一个比较大的质数（1e9+7）。

另外公式\(\ref{eq:hash}\)中在\(h_0\)取模后运算结果会变成负值，而在c++中负数取模还是负数，所以要把减法运算换算成补码运算。 \[ h_1 = (h_0 * b + (M - c_nb^n \mod M) + c_0) \mod M \tag{2} \]

最后，因为取模运算具有类似于分配律的性质，可以用于哈希算式的迭代,即： \[ （x+y）\mod M = (x \mod M + y \mod M) \mod M \\ （x*y）\mod M = ((x \mod M) * (y \mod M)) \mod M \]

// whether nums1 and nums1 has same substring of length len 
bool duplicate(vector<int>& nums1, vector<int>& nums2, int len) {
    unordered_set<long> hcodes1;
    long f = 1;
    long hc = prefixHash(nums1, len, f);
    hcodes1.insert(hc);
    for (int cur = 1; cur <= nums1.size()-len; cur ++) {
        hc = hc * base + (md - nums1[cur-1] * f % md) + nums1[cur + len - 1];
        hc %= md;
        hcodes1.insert(hc);
    }

    long hc2 = prefixHash(nums2, len, f);
    if (hcodes1.count(hc2)) return true;
    for (int cur = 1; cur <= nums2.size()-len; cur ++) {
        hc2 = hc2 * base + (md - nums2[cur-1] * f % md) + nums2[cur + len - 1];
        hc2 %= md;
        if (hcodes1.count(hc2)) return true;
    }

    return false;
}

long prefixHash(vector<int>& nums, int len, long& f) {
    long res = 0;
    f = 1;
    for (int d = len - 1; d >= 0; d --) {
        res = (res + f * nums[d]) % md;
        f = f * base % md;
    }
    return res;
}
